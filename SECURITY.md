# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **do not** create a public GitHub issue. Instead, please report it responsibly to us.

### How to Report

1. Email: silas@alt-heaux.site with subject "SECURITY VULNERABILITY"
2. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (if applicable)

### What to Expect

- We will acknowledge receipt within 48 hours
- We will assess the severity and work on a fix
- We will keep you updated on progress
- We will credit you in the fix (unless you prefer anonymity)
- Fixes will be released as soon as possible

### Security Best Practices

- Keep your dependencies updated
- Never commit secrets or credentials
- Use strong, unique passwords
- Enable 2FA on your GitHub account
- Review all pull requests carefully before merging

### Supported Versions

| Version | Status |
|---------|--------|
| Latest  | Fully supported |
| Older   | Not supported |

Thank you for helping keep this project secure!
